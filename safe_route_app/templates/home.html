Complete Testing Walkthrough

Follow this exact sequence to verify the entire system lifecycle, from police resource allocation to emergency response.

Phase 1: Police Setup & Resource Allocation

Step A: Register User Account

1. Go to the active web app and click Get Started.

2. Register a new account with a police email (e.g., police.indiranagar@karnataka.gov.in).

Step B: Admin Verification (Backend)

1. Log in to the Admin Panel at /admin/.

2. Use superuser credentials: admin / 1234 (or as configured).

3. Navigate to "Police Authorities" and click "Add Police Authority".

4. Select the user created in Step A, assign a Badge Number, Rank, and Station.

5. Save to grant this user police dashboard access.

Step C: Assign Duty & Patrols

1. Log out of Admin and log in to the main app with the police account from Step A.

2. Navigate to /police/dashboard/.

3. Identify high-risk zones on the map and click "Assign Patrol Unit". This activates safety measures for that beat.

Phase 2: User Journey & SOS Trigger

Step A: User Registration

1. Register a new public user account (e.g., citizen@test.com).

2. Login to access the main map interface.

Step B: Route Selection

1. Enter specific Indian locations (e.g., "M G Road" to "Indiranagar").

2. Review the Safety Score. You will see higher scores in areas where you assigned police patrols in Phase 1.

Step C: Trigger Emergency

1. Simulate a threat by clicking the red SOS Button.

2. A countdown will start. Let it finish to send the alert.

3. You will see a "Police Notified" confirmation screen.

Phase 3: Real-Time Response Verification

Step A: Verify Alert Reception

1. Open a new browser window (Incognito) or logout and login back as Police.

2. Check the Dashboard. The SOS alert from Phase 2 should appear instantly with the user's live location.

Step B: Response Coordination

1. Verify that the assigned Patrol Unit (from Phase 1) is visible near the incident.

2. This confirms the complete loop: Simulation -> Prevention -> Action -> Response.
